## 一、recv_msg 实现

当前在 `kernel/src/net/socket/packet/mod.rs` L626-632 返回 ENOSYS。

### 实现步骤
参考 UDP socket 的 recv_msg（kernel/src/net/socket/inet/datagram/mod.rs:1841-2007）：

1. 从 msg.msg_iov/msg.msg_iovlen 创建 IoVecs：
   ```rust
   let iovs = unsafe { IoVecs::from_user(msg.msg_iov, msg.msg_iovlen, true)? };
   let mut buf = iovs.new_buf(true)?;
   ```

2. 用 try_recv 接收数据到 buf（try_recv 返回 `(usize, PacketMetadata)`）

3. Scatter 到用户 iovec：
   ```rust
   iovs.scatter(&buf[..copy_len])?;
   ```

4. 如果 msg.msg_name 不为 null，写 sockaddr_ll：
   - 从 PacketMetadata 构造 sockaddr_ll：
     - sll_family = 17 (AF_PACKET)
     - sll_protocol = metadata.protocol (已是网络字节序)
     - sll_ifindex = metadata.ifindex as i32
     - sll_hatype = 1 (ARPHRD_ETHER)
     - sll_pkttype = metadata.pkt_type as u8
     - sll_halen = 6
     - sll_addr = [metadata.src_mac[0..6], 0, 0]
   - 写回 msg.msg_name，更新 msg.msg_namelen = size_of::<SockAddrLl>()

5. 如果 PACKET_AUXDATA 选项已启用且 msg.msg_control 不为 null，写 tpacket_auxdata cmsg：
   - tpacket_auxdata 结构（参考 Linux include/uapi/linux/if_packet.h）：
     ```rust
     #[repr(C)]
     struct TpacketAuxdata {
         tp_status: u32,
         tp_len: u32,      // 原始帧长度
         tp_snaplen: u32,  // 实际捕获长度
         tp_mac: u16,
         tp_net: u16,
         tp_vlan_tci: u16,
         tp_vlan_tpid: u16;
     }
     ```
   - cmsg level = SOL_PACKET (263), type = PACKET_AUXDATA (8)
   - 用 CmsgBuffer 写入（参考 kernel/src/net/socket/unix/utils.rs 的 CmsgBuffer::put 方法）

6. 设置 msg_flags（如有截断设 MSG_TRUNC）

## 二、send_msg 实现

当前在 L634-640 返回 ENOSYS。

### 实现步骤
1. 从 msg.msg_iov/msg.msg_iovlen 创建 IoVecs
2. Gather 到单个 buffer：
   ```rust
   let buf = iovs.gather(true)?;  // 或 read from user
   ```
   注意：IoVecs 可能有 gather 或类似方法。参考 UDP send_msg。

3. 如果 msg.msg_name 不为 null，解析目标 sockaddr_ll
4. 调用 try_send(buf, dest) 发送

## 三、Multicast Membership (ADD/DROP_MEMBERSHIP)

当前 set_option 中 ADD/DROP_MEMBERSHIP 返回 Ok(()) 不做任何事。

### 实现
1. 定义 packet_mreq 结构体（对应 Linux struct packet_mreq）：
   ```rust
   #[repr(C)]
   pub struct PacketMreq {
       pub mr_ifindex: u32,
       pub mr_type: u32,
       pub mr_alen: u16,
       pub mr_address: [u8; 8],
   }
   ```

2. 定义 mr_type 常量：
   - PACKET_MR_PROMISC = 0 (混杂模式)
   - PACKET_MR_MULTICAST = 1 (特定多播组)
   - PACKET_MR_ALLMULTI = 2 (所有多播)
   - PACKET_MR_UNICAST = 3 (单播)

3. 在 set_option 的 ADD_MEMBERSHIP 分支：
   - 解析 val 为 PacketMreq
   - 根据 mr_type 处理：
     - PACKET_MR_PROMISC: 在指定网卡上启用混杂模式（设置 InterfaceFlags::PROMISC）
     - PACKET_MR_ALLMULTI: 在指定网卡上启用 ALLMULTI（设置 InterfaceFlags::ALLMULTI）
     - PACKET_MR_MULTICAST: 暂时可以存储多播地址列表（完整的多播过滤可后续完善）
   - DROP_MEMBERSHIP 做相反操作

4. InterfaceFlags 定义在 kernel/src/driver/net/types.rs，有 PROMISC 和 ALLMULTI 标志
5. 通过 netns.device_list() 按如果 mr_ifindex 查找网卡，设置/清除 flags
